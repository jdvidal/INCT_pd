Identificando dados com coordenadas correspondentes ao mar

Nesse tutorial aprenderemos a identificar coordenadas fora do continente em dados do SpeciesLink.

Como exemplo, usaremos os registros de Apocynaceae do SpeciesLink (https://specieslink.net/).

Instalao e carregamento dos pacotes

Vamos precisar dos pacotes openxlsx, para ler arquivos Excel, raster, para manusear formatos de arquivos SIG, e CoordinateCleaner para filtrar os registros geogrficos.

  • openxlsx (Schauberger et al., 2020): https://ycphs.github.io/openxlsx/index.html
  • raster (Hijmans, 2021): https://CRAN.R-project.org/package=raster
  • CoordinateCleaner (Zizka et al., 2019): https://github.com/ropensci/CoordinateCleaner

Instalamos os pacotes com o comando install.packages():

install.packages(c("openxlsx", "raster", "CoordinateCleaner"), repos="http //cran.us.r-project.org")

Carregamos os pacotes com o comando library():

library("openxlsx")
library("raster")
library("CoordinateCleaner")

Setup do ambiente de trabalho e carregamento dos dados

Vamos definir o diretrio de trabalho, onde estar a planilha baixada do SpeciesLink. Para isso, usamos a funo set.wd().

setwd("/Users/jdvidal/Desktop/INCT/")

Vamos usar a funo read.xlsx() e read.csv() para ler a planilha.

records <- read.xlsx("/Users/jdvidal/Desktop/INCT/speciesLink-20210920072313-0004598.xlsx")

E a funo shapefile para ler o shapefile com os limites geogrficos e nomes dos pases do mundo.

countries <- shapefile("/Users/jdvidal/Desktop/INCT/ne_10m_admin_0_countries/ne_10m_admin_0_countries.shp")

Aqui temos nosso primeiro problema.

class(records$latitude)
## [1] "character"

Os valores de coordenadas na coluna records$latitude esto sendo interpretados como texto. Para convert-los para nmeros, usamos a funo as.numeric().

records$latitude <- as.numeric(records$latitude)
records$longitude <- as.numeric(records$longitude)

Vamos nos certificar de usar apenas dados que possuam informao geogrfica. Para isso, usamos complete.cases() para selecionar apenas linhas cujos valores esto completos para a coluna latitude.

records <- records[complete.cases(records$latitude),]

Vamos extrair os pontos que caem no mar com o CoordinateCleaner. Usamos a funo cc_sea() para remover coordenadas que carem no mar. Essa funo leva trs argumentos, sendo o primeiro deles o objeto data.frame com as ocorrncias, o segundo argumento lon referente ao nome da coluna contendo os valores de longitude e o terceiro argumento lat, cujo valor corresponde ao nome da coluna contendo os valores de latitude.

records_filtered <- cc_sea(records, lon = "longitude", lat="latitude")

list_sea_mismatches <- records[!records$catalognumber%in%records_filtered$catalognumber,]
nrow(list_sea_mismatches)
## [1] 71

Plotando os pontos em um mapa

Vamos agora converter o data.frame records para um objeto espacial. Para isso, usamos a funo coordinates() e declaramos que as colunas longitude e latitude correspondem aos valores de coordenadas x e y com o argumento ~longitude+latitude.

coordinates(records) <- ~longitude+latitude
coordinates(list_sea_mismatches) <- ~longitude+latitude

Uma vez atribudas as coordenadas, podemos plotar os dados problemticos com a funo points. Vamos usar o shapefile countries como base. Como queremos apenas a Amrica do Sul no nosso mapa, vamos selecionar dentro do shapefile countries os componentes cujo campo CONTINENT seja igual a "South America".

plot(countries[countries$CONTINENT=="South America",])
points(records, cex=0.5)
points(list_sea_mismatches, cex=0.5, col="red")

Existem 71 dados com coordenadas caindo no oceano. Ns podemos corrigir estes valores individualmente atravs das etiquetas originais.

Por fim, salvamos estes dados num arquivo xlsx com a funo write.xlsx().

write.xlsx(list_sea_mismatches, "/Users/jdvidal/Desktop/INCT/Results_ocean_records.xlsx")
